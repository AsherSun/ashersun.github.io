# 什么是Git?

git介绍

git是什么？
Git是目前世界上最先进的分布式版本控制系统（没有之一）。
Git有什么特点？
简单来说就是：高端大气上档次！
那什么是版本控制系统？
简单来说：第一天写了一篇文章保存下来。但有些地方不太满意，第二天把第一天的文章复制一份，修改下。第三天的时候又有了新的想法，再把第二天的文章复制一份修改下。这时有了三个版本也就是三个文件。而git只需要再一个文件中修改，但是三次的改动都会有记录，这就是版本控制系统

来自廖雪峰-git教程 (opens new window)

# git常用命令

这里总结初次接触git会常用的命令

  • git init: 初始化一个git仓库
  • git add .: 将目前工作目录中的所有修改文件添加到暂存区
  • git status: 查看当前工作目录中是否有修改的文件
  • git commit -m '当前修改的一段描述': 对暂存区的文件添加一段当前修改的描述并且当这次修改的记录保存下来
  • git push origin master: 将本地git仓库中的修改推送到远程仓库
  • git branch: 查看本地仓库的git 分支
  • git branch -D fix/swiper: 删除本地fix/swiper分支
  • git checkout -b feat/change-address: 切换并且创建一个新的分支，这个分支的名字叫做feat/change-address
  • git checkout feat/change-phone: 切换到已有的分支,分支的名称叫做chagne-phone

上面的一些命令在日常工作中是最常用的。后续的一些工作场景中会使用的命令，还请看下廖雪峰的git教程。这里有一份git命令备份 (opens new window)

# git commit规范

[type]([scope]): #[issue-number]{.issue-part} [subject]
1

type的标识

type 描述
feat 新增功能
fix 修补bug
docs 文档
style 格式(不影响代码运行的变动)
refactor 重构(即不是新增功能，也不是修改bug的改动)
test 测试
chore 构建过程、辅助工具的变动
perf 应用性能优化，不是优化(重构)代码

scope 用于说明更改范围，比如：数据层、视图层、控制层 subject 是对当前改动的一段简短描述，不超过50个字符

  2. 进入新建仓库页面，填写仓库名称，格式为：username.github.io。之后点击创建仓库

  3. 创建成功后点击右上角的设置，进入设置页面。

  4. 找到 Github Pages选项，选择设置使用GitHub Pages
