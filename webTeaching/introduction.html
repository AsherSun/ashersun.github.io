# 什么是web?

web定义

web（World Wide Web）即全球广域网，也称为万维网，它是一种基于超文本和HTTP的、全球性的、动态交互的、跨平台的分布式图形信息系统

来自百度百科 (opens new window)

# 什么是web前端？

web前端

# web前端是做什么的？

通俗点讲，web前端是负责处理用户交互的，包括浏览器中的网页、手机APP界面、PC端应用界面、智能售货机的操作界面、各种终端需要展示的界面都可以由前端工程师来处理

# web前端是从计数机问世就存在的职位吗？

在20005年之前是没有前端开发这个名词的。那时候处理浏览器页面展示大多数由美工、或者后端工程师负责。

直到Ajax的出现，出现了一批专门以js为业的程序员

前端的发展从网页的展示特效、交互特效到用户体验再到前后端分离，之后就是性能、模块化、工程化、多端运行

# web前端技术发展历史

# 诞生

Web 刚诞生的使命是用户科研人员的资料间共享

  1. 1990年，以超文本语言HTML为基础出现了最原始的Web浏览器
  2. 1991年，Web理念广泛推广，美国国家超算应用中心开发了名为Mosaic浏览器
  3. 1994年，设立HTML工作组；并出现了Mosaic Netscape 1.0 beta 浏览器，后改名为Navigator；万维网联盟（W3C）成立，标志万维网正式诞生
  4. 此时的网页以HTML为主，是纯静态网页，信息流只能通过服务器到客户端单项流通，仅为Wab1.0时代
  5. 1995年，出现JavaScript语言，网景公司将这种脚本语言嵌入进Navigator2.0，使其能在浏览器中运行
  6. 1996年，微软发布VBScript和JScript，并内置于 Internet Explorer 3。但 JavaScript 与 JScript 两种语言的实现存在差别，这导致了程序员开发的网页不能同时兼容 Navigator 和 Internet Explorer 浏览器，导致第一次浏览器战争。
  7. 1996年W3C推出CSS1.0规范
  8. 1997年，HTML3.2作为W3C推荐标准发布；ECMA以JavaScript语言为基础制订了ECMAScript1.0标准规范，自此浏览器厂商开始逐步使用ECMAScript规范
  9. 1997年，HTML4.0规范发布
  10. 1998年，W3C推出CSS2.0规范；ECMAScript 2 规范发布，并通过ISO生成正式国际标准
  11. 1999年，ECMAScript3规范发布，并成为主流浏览器最广泛使用和实现的语言规范基础

# 浏览器厂商间的战争

浏览器厂商间的战争，促使该技术的快速发展。虽然因为各个厂商所使用的各种标准、协议导致开发人员为了兼容不同浏览器上的显示效果而欲生欲死，但是他们却是整个技术迭代中最重要的力量。

  1. 1995年以前，网景是互联网浏览器的绝对标准
  2. 1995年8月微软开始发布IE 1.0
  3. 1997年10月份发布的IE 4.0浏览器震惊市场
  4. 1998年网景以48亿美元的价格出售给了AOL公司
  5. 1998年网景公开了它的浏览器源码，并重新命名为Mozilla
  6. 第一次浏览器战争以IE浏览器完胜而结束

# 第二次浏览器战争

第一次战争，以IE完胜而结束，而第二次战争也是因为IE的懒惰、傲慢、不作为等开始了。至此，IE走下神坛、江湖上只剩传说

  1. 2002年Mozilla发布了第一个版本
  2. 2004年，基于Mozilla源码的FireFox（火狐）浏览器出世，由开源社区贡献，得到了人们的认可，迅速占领了百分之二十的市场
  3. 随后基于不同引擎的浏览器加入混战

# 技术发展历程

随着浏览器间的战争，浏览器厂商为了扩张市场份额，工程师为了解决高效工作，进而促生了技术的迭代

  • 为了让网页更富展示形式，以 PHP、JSP、ASP.NET 为代表的动态页面技术相继诞生
  • Ajax的诞生，让前端开发可以直接与后端工程师通讯，而不用刷新网页
  • 为了解决各个浏览器间的差异性，DOM对象操作代表性作品：jQuery
  • 顺应时代，以移动优先为理念的响应式技术 jQuery + Bootstraps
  • 进一步提升前端开发效率而出现的前端框架：vue、React、angularjs
  • 前端模块化、工程化：webpack
  • 国内的小程序技术：各大APP平台提供的运行环境、多端编译小程序框架：TaroJs、UniAPP
  • 各种移动端APP解决方案：React Native 、vue weex 、Flutter
  • PC端应用开发解决方啊：Electorn
