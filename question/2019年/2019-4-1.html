# 问：

  1. iconfont 图标在iPhone8 Plus下会出现图标缺失一块(被咬一口)的现象。

  2. overflow:scroll; 在安卓机下部分机型会失效 (oppo R9S)

  3. 移动端input输入框type类型设置number, maxleng会无效

  4. 如何避免项目开发中的css冲突

# 答：iconfont 图标在iPhone8 Plus下会出现图标缺失一块(被咬一口)的现象。

造成这种现象的原因可能是图标本身的四周像素会有缺失的问题。所以在iPhone8 Plus 下渲染会有缺失。其他机型下渲染并无异常

# 解：

  • 解决方案一：如果所有机型都会出现图标被咬一口的情况，可以与UI沟通，提供标准图标(这种出现的可能性极低)
  • 解决方案二：如果只有部分机型出现图标被咬一口的情况，其他机型可以正常显示iconfont。则可以针对部分机型进行媒体查询。使用图片代替。图片四周让UI请预留一像素空白，以防止出现被咬一口的情况。
  • 解决方案三：使用 SCSS mixins 封装 移动端 二倍图、三倍图媒体查询。不推荐这种方式(会占用大量静态资源大小)

解决方案二的代码：

.icon-wrap {
  ...
  .img-icon {
    display: none;
  }
  .icon-svg {
    ...
  }
  // 针对iPhone6/7/8 Plus 进行媒体查询。将iconfont 替换成 background-image
  @media only screen and (device-width: 414px) and (device-height: 736px) and (-webkit-device-pixel-ratio: 3) {
    .icon-svg {
      display: none;
    }
    .img-icon  {
      display: block;
      ...
    }
    .icon-dingdan {
      background-image: url('...');
    }
    .icon-shouhuodizhi {
      background-image: url('...');
    }
    .icon-chuangzuozhongxin {
      background-image: url('...');
    }
  }
}

# 答：overflow:scroll; 在安卓机下部分机型会失效 (oppo R9S)

造成这种现象的原因：有 overflow:scroll; 样式元素的父元素有 overflow:hidden;属性。所以在 oppo R9s下会出现滚动失效的情况

# 解：

  • 解决方案一：将父元素的overflow:scroll;去掉
  • 解决方案二：使用js控制元素滚动

# 答：移动端input输入框type类型设置number, maxleng会无效

如果 **type** 的值是 text, email, search, password, tel, 或 url，那么这个属性指明了用户最多可以输入的字符个数（按照Unicode编码方式计数）；对于其他类型的输入框，该属性被忽略。它可以大于 size 属性的值。如果不指定这个属性，那么用户可以输入任意多的字符。如果指定为一个负值，那么元素表现出默认行为，即用户可以输入任意多的字符。本属性的约束规则，仅在元素的 value 属性发生变化时才会执行。译者注:ie10+。 [引用地址](https://developer.mozilla.org/zh-CN/docs/Web/HTML/Element/input)

# 解：

原生JS

<input type="number" oninput="if(value.length>5)value=value.slice(0,5)" />
1

Vue

<template>
	<input v-model="userPhone" />
</template>

<script>
	export default {
		name: 'testMaxlength',
		data() {
			return {
				userPhone: ''
			}
		},
		watch: {
			userPhone(newValue) {
				this.userPhone = newValue.slice(0, 11)
			}
		}
	}
</script>

# 答：如何避免项目开发中的css冲突

  1. 单一组件的根元素要具有独立 className 名称（请确保在项目中的唯一性）

  2. 单一组件根元素可以携带唯一标识符, 比如 sass-bem-constructor 工具、

  3. 团队中规范开发规范，引入CSS规范，比如：BEM、OOCSS 等

# 解：

  • 单一组件根元素可以携带而外标识符

对于组件库开发，唯一标识符会是组件库名称的缩写。比如：有赞 van-xxx、饿了吗 el-xxx

对于项目开发，唯一标识符可以用于分类。比如：页面级组件可以携带：page-xxx、通用组件可以用 components的首字母缩写 c

sass-bem-constructor 工具则是遵循BEM规范，标识符为BEM中分类的 “块”、“修饰符”、“状态”等

  • 团队中规范开发规范
    • BEM (opens new window)
    • OOCSS (opens new window)
    • 网易 (opens new window)
